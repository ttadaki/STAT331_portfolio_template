The Power of Efficiency

Efficiency in a general sense refers to the idea of minimizing the amount of work that is required to complete a goal or task. In the context of statistical computing for this course, this primarily means optimizing one's code by reducing the amount of steps and computations it takes to reach a desired result. This saves time and energy as it takes less memory and thus time in order to run the code as well as reduces the risk of errors. This can be achieved in a variety of ways. In this course specifically, I learned how to write efficiently through the practice of being concise as well as the use of functions, iteration, and modern tools.

First, the use of functions can greatly improve efficiency by allowing one to produce code that can be reused. Rather than writing the same code over and over again, one can call the function with the proper arguments to achieve the same goal.

Similarly, iteration can allow for the same task to be performed on a number of data points at once without having to repeat the code. We primarily saw this through the use of the across function, which allowed us to apply a function or operation to multiple columns at once, and the map function, which allowed us to apply a function to a specified number of elements in a vector or list.

Finally, modern tools like the dplyr package allow us to produce efficient code as they were specifically designed with efficiency and functionality in mind. When tools are developed or updated, the professionals producing them have the intent of making R as a whole easier and more optimal to use. My personal favorite tool that we have used in this class is the piping operator. This operator is used to intuitively chain function calls and outputs together in order to make code easier to read and understand. In Lab 4, the first assignment in which it was used, the piping operator was difficult to understand, as I essentially had to learn how to read the code backwards from how I was doing before, but with time, it soon became my method of choice for how much easier it became to follow and edit my code. 

The moment in which I fully understood how significant efficient code was in Lab 8, when we used the map function to iterate through the entire song using the functions we made to write each line of the song. By using functions, we were able to compose each individual line based on the arguments given. By using the mapping function, we could use the elements inside our dataframe by passing each column into argument to build each unique line of the song. I realized that by using these tools, I was not only saving time by not having to rewrite lines of code, but also saving memory on my computer and reducing the time it would take to run the code.

Overall, writing concise code demonstrates a deep understanding of the code at hand while also improving the readability and efficiency. Code that is too long or convoluted can be difficult to maintain and modify. It also makes it more difficult to debug, as errors will be more difficult to identify and fix. In general, long code takes more memory and lengthens the time necessary to run, which in today's computation heavy environment, are key points that we are trying to optimize.
